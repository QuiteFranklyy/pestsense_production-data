### About Sensahub
**Sensahub** (by Sensavation Pty Ltd) is an *application enablement software platform* that **accelerates the development of sophisticated applications** for web and mobile by integrating IoT functionality with existing business systems, devices and web services. ​

**Sensahub** can be used in many diverse situations including:​

+ **Monitoring** and **controlling** external systems.​

+ **Vertical business applications** that use IoT data.​

+ **Integrating**, **processing** and **analysing** disparate data sources.​

+ **Mobile** and **web** versions of IoT applications. ​

+ **Forms** and **workflow** for field data collection.​

> Sensahub has the flexibility to accommodate your specific business requirements for customised IoT solutions​.

---

### Addressing IoT Concerns

Sensahub aims to **remove some of the complexity** from designing, building, deploying and using IoT solutions.​

Sensavation believes the **value in IoT solutions** lies mostly in the ability to use Sensahub as the basis for business applications that integrate IoT data streams with existing systems and data services.​

Sensahub’s focus is an **application builder for IoT business solutions**, to enable businesses to **realise benefit** from IoT without requiring significant skills, time or funding.​

### Sensahub Ecosystem
A Sensahub solution comprises of **connections to the outside world**, a **server** for processing the data, and **clients** for the end user application. ​

Sensahub can be run on standalone, edge or on-premise systems as small as a **Raspberry Pi** up to an **enterprise class server**.​

For **cloud solutions**, Sensahub can be deployed on both private and public cloud (SaaS) based systems. ​

The user **interacts** with the Sensahub client **via a browser application** customised for a wide range of devices (with mobile optimisations).​

<div class="row">

##### External Connections
+ Connect Sensahub to the outside world through Sensahub plugin connectors.

+ Handle the network, security, and data format requirements of the remote end.

+ Automatically convert incoming data into standardised Sensahub formats.

</div>

<div class="row">

##### Server Processing
+ Transform the incoming data through a graphical flow programming paradigm.

+ Use functional nodes and scripts to apply calculations, conditions, branching, and parsing of data streams.

+ Transform data node by node as defined by links between function nodes.

</div>

<div class="row">

##### Client Applications
+ Use graphical widget elements to build application screens for mobile and web.

+ Display and interact with data sent to the clients by the server using a channel based communication system.

+ Use events and scripts to orchestrate sophisticated layouts and workflows.
</div>
